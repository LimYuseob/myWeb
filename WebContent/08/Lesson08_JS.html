CSS(Cascading Style Sheet) 의 사용 목적 :
웹문서의 내용과 상관없이 디자인 적용 --> 내용을 만약 내용을 수정 하여도 디자인에 영향없음
반응형 Web 구성 --> 사용자의 접근 방법과 상관없이 브라우저의 크기에 따라 레이아웃을 자동으로 변경함.

Style 에 대하여

  • Style 형식(기본 type) : 선택자 {속성1 : 값1; 속성2 : 값2;} ex> --> p{text-align : center; color : blue;}
  • Enter 를 이용한 라인 개행은 스타일에 영향을 미치지 않는다.
  • /**/ 을 이용하여 주석을 처리한다.
  • Style Sheet 의 종류
  • 브라우저 기본 스타일 : Tag 자체에 적용된 스타일 시트 적용
  • 사용자 스타일
    1. 인라인 스타일 시트 : 특정 Element 에 스타일 적용 --> ex>

      이건 인라인 스타일시트.

    2. 내부 스타일 시트 : 웹문서와 CSS 를 같은 문서내에 정의 한 경우.
      <head>
      <style>
      Style 정의..
      </style>
      </head>

      내부 시타일시트 적용시 이렇게 보입니다..

    3. 외부 스타일시트 : 내부CSS 는 중복의 문제 발생..따라서 모든 Style 을 정의한 CSS 파일을 두고 import 해서 사용하는 경우를 말함.
      Head Tag 사이에 import 선언 --> <link rel="stylesheet" href = "src 경로">

CSS 기본 선택자(Selector) : CSS 규칙이 적용될 요소를 지정하는 것.

  • * --> 웹 문서의 모든 선택자를 지정함. 일반적으로 시트일 초기화시 사용함.
  • 요소 선택자 : Tag Name {Style 규칙} : 타입 선택자(모든 요소의 Style 적용)
  • 클래스 선택자 : 특정 부분에 스타일 적용시 사용. 같은 태그라도 다른 스타일을 사용시 특정 부분만 클래스 선택자를 적용한다.
    클래스 선언은 반드시 "classNmae{스타일 규칙(Rule set)}"
    Tag 속성중 class 속성의 값으로 클래스명을 사용한다. 하나의 Element 내에 2개 이상 적용가능하다.
    ex-->

    두개의 클래스를 적용한 예

  • id 선택자 : 특정 부분에 한 번만 적용가능한 선택자. #ID{Rule Set} 으로 정의하며, 문서에서 한번만 사용 가능 하므로 주로 문서의 레이아웃과 관련된 스타일이나, 자바스크립트에서 요소 구별시 사용된다.
  • ex>

  • 그룹선택자 : 같은 스타일을 사용하는 요소 Group화 "," 를 사용하여 선택자를 나열한다. ex>h1,p {Rule Set}

Cascading : 우선순위에 관련된 기준. 기본적으로 위에서 아래로의 우선순위가 결정됨. 만약 웹 요소에 둘이상이 적용시 우선순위에 따라 스타일이 결정됨.

우선순위

  1. 사용자 스타일 : 웹브라우저나, OS 의 설정에서 적용된 Style
  2. 제작자 스타일 : Designer 등의 Style
  3. 사용자 스타일 : 웹브라우저 기본 Style

적용범위

  1. !important : 최우선 적용
  2. Inline CSS
  3. id Style
  4. class Style
  5. type Style
